# Link in Bio v0.2.0: Dynamic Blog Integration with Blogverse API

We're thrilled to announce the release of **Link in Bio v0.2.0**, a significant update that transforms how your blog content is displayed in your portfolio. This version introduces seamless **Blogverse API integration**, eliminating the need for hardcoded blog data and bringing your content to life automatically.

## What's New in v0.2.0

### Live Blog Integration via API

The most impactful feature in this release is the integration of the **Blogverse API**. Your Link in Bio portfolio now fetches your latest blog posts directly from your Blogverse instance in real-time.

**API Endpoint:**
```
https://blogverse-five-omega.vercel.app/api/v1/posts/recent?limit=5
```

This means:
- **No more hardcoding** blog post information
- **Automatic updates** whenever you publish new content on Blogverse
- **Live carousel** that reflects your current blog posts
- **Zero configuration** required after setup

> **Why This Matters:** Previously, every time you published a new blog post, you had to manually update the `blogData` array in `app/page.tsx`. Now, that process is completely automated. Your portfolio instantly reflects your latest content.

### How It Works

The updated carousel component now:

1. **Fetches data** from your Blogverse instance on component mount
2. **Transforms the response** to match the carousel's internal structure
3. **Renders posts dynamically** with proper styling and animations
4. **Handles errors gracefully** with fallback messaging
5. **Displays real blog thumbnails** when available, skips images for blogs without covers

**Behind the Scenes:**

```typescript
useEffect(() => {
  const fetchBlogs = async () => {
    try {
      const response = await fetch(
        "https://blogverse-five-omega.vercel.app/api/v1/posts/recent?limit=5"
      )
      const data = await response.json()
      
      // Transform API data to carousel format
      const transformedBlogs = data.map((post, index) => ({
        id: index + 1,
        title: post.frontmatter?.title || "Untitled",
        description: post.frontmatter?.description || post.excerpt || "",
        slug: post.slug,
        thumbnail: post.frontmatter?.cover || null,
        category: post.frontmatter?.tags?.[0] || "General",
        duration: calculateDuration(post.readingTime),
      }))
      
      setBlogs(transformedBlogs)
    } catch (error) {
      console.error("Error fetching blogs:", error)
    }
  }
  
  fetchBlogs()
}, [])
```

## Updated Features in the Carousel

### Conditional Thumbnail Rendering

v0.2.0 includes intelligent thumbnail handling:

- **Shows thumbnails** only when blog posts have a `cover` image from Blogverse
- **Skips images** for blogs without cover images to maintain clean UI
- **Displays badges** for category and reading duration
- **Smooth hover effects** with play button overlays
- **Responsive sizing** across all device types

### Direct Blog Navigation

Clicking on a blog post now:
- Opens the full blog post in a new tab
- Links directly to `https://blogverse-five-omega.vercel.app/blog/{slug}`
- Removes the modal popup for cleaner user experience
- Maintains carousel auto-play functionality

### Carousel Enhancements

The carousel maintains all beloved features while adding new capabilities:

- **Auto-advancing slides** every 5 seconds
- **Manual navigation** with previous/next buttons
- **Touch/swipe gestures** on mobile devices
- **Dot indicators** for quick navigation
- **Dynamic dot count** based on fetched posts
- **Smooth animations** with Framer Motion

## Installation & Migration

### Upgrading from v0.1.0

If you're upgrading from the previous version:

1. **Pull the latest changes**
   ```bash
   git pull origin main
   ```

2. **Or checkout the new version**
   ```bash
   git checkout v0.2.0
   ```

3. **Install dependencies** (no new dependencies required)
   ```bash
   npm install
   ```

4. **Update your Blogverse URL** in `app/page.tsx` if your instance URL differs:
   ```typescript
   const response = await fetch(
     "https://your-blogverse-instance.com/api/v1/posts/recent?limit=5"
   )
   ```

5. **Restart development server**
   ```bash
   npm run dev
   ```

### Using v0.1.0 (Hardcoded Blogs)

If you prefer the hardcoded approach:

- **Checkout the release tag:** `git checkout v0.1.0`

## Configuration Options

### Customising the Blog API

You can modify the API request to fetch different numbers of posts:

```typescript
// Fetch 10 most recent blogs instead of 5
const response = await fetch(
  "https://blogverse-five-omega.vercel.app/api/v1/posts/recent?limit=10"
)

// Fetch posts by specific tag
const response = await fetch(
  "https://blogverse-five-omega.vercel.app/api/v1/posts/tag/Next.js"
)
```

### Environment Variables

For custom Blogverse instances, add to `.env.local`:

```env
NEXT_PUBLIC_BLOGVERSE_URL=https://your-blogverse-instance.com
NEXT_PUBLIC_BLOG_LIMIT=5
```

Then update the fetch call:

```typescript
const BLOGVERSE_URL = process.env.NEXT_PUBLIC_BLOGVERSE_URL || 
  "https://blogverse-five-omega.vercel.app"
const BLOG_LIMIT = process.env.NEXT_PUBLIC_BLOG_LIMIT || 5

const response = await fetch(
  `${BLOGVERSE_URL}/api/v1/posts/recent?limit=${BLOG_LIMIT}`
)
```
