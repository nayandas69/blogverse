## What is Auto Website Visitor?

Auto Website Visitor represents a meticulously engineered Python automation tool designed to simulate authentic website visits through real browser instances. Unlike rudimentary scripts that merely send HTTP requests, AWV leverages Selenium WebDriver technology to execute genuine browser sessions, thereby replicating the behaviour of actual human visitors with remarkable accuracy.

The tool is currently at version **3.0.0** and has been thoughtfully architected to accommodate both novice users seeking simplicity and advanced practitioners requiring granular control over their automation workflows.

## Key Features and Capabilities

### Multi-Browser Support

One of AWV's most distinguishing characteristics is its comprehensive support for multiple web browsers. The tool seamlessly integrates with:

- **Google Chrome** - The world's most popular browser
- **Mozilla Firefox** - A privacy-focused alternative
- **Microsoft Edge** - The modern Chromium-based browser

This flexibility ensures that users can simulate traffic from diverse browser environments, which proves particularly valuable when testing cross-browser compatibility or analysing browser-specific behaviours.

### Intelligent Scheduling System

AWV incorporates a sophisticated scheduling mechanism that supports two distinct scheduling paradigms:

**Interval-based Scheduling** allows users to specify time intervals using intuitive notation:

```bash
awv --url https://example.com --schedule "1h"   # Every hour
awv --url https://example.com --schedule "30m"  # Every 30 minutes
awv --url https://example.com --schedule "45s"  # Every 45 seconds
```

### Headless Operation Mode

For server deployments or scenarios where visual browser rendering is unnecessary, AWV supports headless operation. This mode executes browser automation without displaying the graphical interface, significantly reducing resource consumption whilst maintaining full functionality.

### Auto-Scroll Functionality

To simulate authentic user engagement, AWV includes an intelligent auto-scroll feature that progressively scrolls through web pages, mimicking the natural reading patterns of human visitors. This capability proves essential when dealing with lazy-loaded content or infinite scroll implementations.

### Proxy Support and Custom Headers

For users requiring anonymity or geographical flexibility, AWV provides comprehensive proxy support, including authenticated proxy configurations:

```bash
awv --url https://example.com --proxy "192.168.1.1:8080"
awv --url https://example.com --proxy "username:password@192.168.1.1:8080"
```

Additionally, custom headers can be configured through environment variables, enabling sophisticated request manipulation.

## Installation and Setup

Installing AWV is remarkably straightforward, requiring merely a single command:

```bash
pip install auto-website-visitor
```

Following installation, the tool becomes immediately accessible through the `awv` command-line interface.

### System Requirements

- Python 3.8 or higher
- One of the supported browsers installed on your system
- Adequate system resources for browser automation

## Usage Patterns

### Interactive Mode

For users who prefer a guided experience, AWV offers an interactive mode that presents a menu-driven interface:

```bash
awv --interactive
```

This mode presents the following options:

1. **Configure and run visitor** - Step-by-step configuration wizard
2. **Load configuration file** - Import existing configuration
3. **Save current configuration** - Export settings for future use
4. **Check for updates** - Verify and install latest version
5. **Exit** - Terminate the application

### Command-Line Interface

For scripting and automation purposes, AWV provides a comprehensive CLI:

```bash
# Basic usage
awv --url https://example.com --count 10 --interval 5

# Advanced usage with multiple options
awv --url https://example.com \
    --count 100 \
    --interval 10 \
    --browser firefox \
    --headless \
    --auto-scroll \
    --random-delay

# Using configuration file
awv --config /path/to/config.yaml
```

### Configuration Files

AWV supports both YAML and JSON configuration formats, enabling users to maintain persistent settings:


To generate a sample configuration file:

```bash
awv create-config --config-path ./my-config.yaml
```

### Programmatic Usage

For developers seeking to integrate AWV into larger applications, the tool exposes a clean Python API:

```python
from auto_website_visitor import AutoWebsiteVisitor, VisitorSettings

# Configure settings
settings = VisitorSettings(
    url="https://example.com",
    visit_count=10,
    interval=5,
    browser="chrome",
    headless=True,
    auto_scroll=True
)

# Execute visits
visitor = AutoWebsiteVisitor(settings)
success = visitor.run()

# Retrieve statistics
stats = visitor.get_stats()
print(f"Success rate: {stats['success_rate']:.1f}%")
```

## Architecture Overview

AWV employs a modular architecture comprising several interconnected components:

### Core Module

The `AutoWebsiteVisitor` class orchestrates the entire visiting process, managing browser instances, implementing retry logic, and collecting execution statistics.

### Browser Manager

The `BrowserManager` component handles browser instantiation, WebDriver configuration, and the execution of page interactions including navigation, scrolling, and timeout management.

### Configuration System

The `Config` class provides a unified interface for loading, validating, and persisting configuration settings from multiple sources including command-line arguments, environment variables, and configuration files.

### Scheduler

The `SchedulerManager` implements both interval-based and cron-based scheduling using dedicated threading for non-blocking execution.

### Logger

A comprehensive logging system provides detailed execution feedback with support for log rotation, multiple output destinations, and configurable verbosity levels.
