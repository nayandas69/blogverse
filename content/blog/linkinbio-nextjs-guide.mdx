# Link in Bio: Building a Modern Portfolio with Next.js and Glassmorphism

In the contemporary digital landscape, establishing a compelling online presence has become indispensable for content creators, developers, and professionals alike. The **Link in Bio** project represents a sophisticated solution to this challenge, offering a visually stunning portfolio built with **Next.js 14** that seamlessly consolidates all your social media profiles and content into a single, elegant interface.

## Understanding the Project's Vision

The Link in Bio portfolio transcends the conventional notion of a simple landing page. It embodies a meticulously crafted digital showcase that employs cutting-edge design principles, most notably **glassmorphism**, to create an immersive user experience. This aesthetic approach, characterised by translucent surfaces, subtle blur effects, and sophisticated layering, has gained considerable traction in modern web design for its ability to convey depth and elegance simultaneously.

> **What is Glassmorphism?** This design trend creates a "frosted glass" effect using background blur, transparency, and subtle borders. It provides visual hierarchy whilst maintaining a clean, modern aesthetic that works beautifully across both light and dark themes.

### Three-State Theme System

Unlike conventional binary theme toggles, this project implements a sophisticated three-state system:

```typescript
const toggleTheme = () => {
  let newTheme: "light" | "dark" | "system"
  
  if (themeMode === "light") {
    newTheme = "dark"
  } else if (themeMode === "dark") {
    newTheme = "system"
  } else {
    newTheme = "light"
  }
  
  setThemeMode(newTheme)
  setCookie("theme-preference", newTheme)
}
```

This approach allows users to choose between light mode, dark mode, or automatic system preference detection, with their choice persisting across browser sessions via cookies.

## Installation and Setup

### Prerequisites

Before commencing installation, ensure your development environment meets these requirements:

- **Node.js** version 18.0 or higher
- **npm** version 8.0 or higher

### Step-by-Step Installation

1. **Clone the repository**

```bash
git clone https://github.com/nayandas69/linkinbio-nextjs.git
cd linkinbio-nextjs
```

2. **Install dependencies**

```bash
npm install
```

3. **Configure environment variables**

```bash
cp .env.example .env.local
```

Then edit `.env.local` with your configuration:

```bash
# Google Search Console Verification
GOOGLE_VERIFICATION_CODE=your-verification-code

# Optional configurations
NEXT_PUBLIC_SITE_URL=https://yourdomain.com
NEXT_PUBLIC_AUTHOR_NAME=Your Name
```

4. **Add your images**

Place the following images in the `public/images/` directory:

- `profile.jpg` - Your profile photograph
- `favicon.jpg` - Website favicon
- `blog1.png`, `blog2.png`, `blog3.png` - Blog post thumbnails

5. **Start the development server**

```bash
npm run dev
```

6. **Access the application**

Navigate to [http://localhost:3000](http://localhost:3000) in your browser.

> **Important:** Ensure all required images are placed in the correct directory before starting the development server to avoid missing asset errors.

## Accessibility Features

The project demonstrates exemplary attention to accessibility standards:

- **Semantic HTML structure** throughout the codebase
- **ARIA labels** on all interactive elements
- **Keyboard navigation** support
- **Screen reader compatibility**
- **Skip to main content** link for keyboard users
- **Reduced motion preferences** respected via CSS media queries
- **High contrast mode** support for users with visual impairments

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

> **Accessibility Tip:** Always test your customised version with screen readers and keyboard-only navigation to ensure all users can access your content effectively.
